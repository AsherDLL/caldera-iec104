# IEC 104 Plugin

The IEC 104 plugin emulates adversary actions over **IEC 60870-5-104**
(IEC-104), the TCP/IP telecontrol protocol (port 2404) used between control
centers and outstations (RTUs/PLCs) in electric power and other utilities.

All abilities call a single generic master payload, `iec104_cli`, which speaks
IEC-104 to any conforming outstation. Targets are parameterized entirely through
facts, so the same abilities work against any device.

## Command reference

The Linux executor is shown; Windows uses `iec104_cli.exe`, macOS uses
`iec104_cli_darwin`.

| Ability | Command |
|---|---|
| Station Interrogation | `iec104_cli #{iec104.server.ip} --port #{iec104.server.port} --ca #{iec104.common_address} interrogate` |
| Counter Interrogation | `iec104_cli ŌĆ” counter-interrogate` |
| Single Command | `iec104_cli ŌĆ” single-command #{iec104.command.ioa} #{iec104.command.value}` |
| Double Command | `iec104_cli ŌĆ” double-command #{iec104.command.ioa} #{iec104.command.value}` |
| Set Point Command | `iec104_cli ŌĆ” setpoint #{iec104.setpoint.ioa} #{iec104.setpoint.value}` |
| Clock Synchronization | `iec104_cli ŌĆ” clock-sync` |

The payload's `single-command`/`double-command`/`setpoint` actions also accept
`--select` (select-before-operate) and `setpoint` accepts `--type na|nb|nc`; add
these to an ability command when the target requires them.

## Facts

| Fact | Description |
|---|---|
| `iec104.server.ip` | outstation IP address / hostname |
| `iec104.server.port` | IEC-104 TCP port (2404) |
| `iec104.common_address` | ASDU common address of the target station |
| `iec104.command.ioa` | information object address for single/double commands |
| `iec104.command.value` | `ON` or `OFF` (quote the value in YAML) |
| `iec104.setpoint.ioa` | information object address for set-point commands |
| `iec104.setpoint.value` | set-point value (number) |

## Payload

The `iec104_cli` payload is built and released from
[`caldera-iec104-payloads`](https://github.com/AsherDavila/caldera-iec104-payloads).
It wraps the `c104` library and returns exit code `0` when the outstation confirms
the request (ACT-CON positive), `1` when there is no positive confirmation, and
`2` on a connection or argument error. The Linux binary targets glibc >= 2.31.
